# Architecture вҖ” Master Gift/Discount Card System

A multi-business platform where a platform owner issues **Master Cards** (themed
discount cards) that bundle many participating businesses (vendors). Customers
add a card to Apple/Google Wallet and redeem discounts in-store via QR / NFC /
manual code. Vendors manage their own discount fields and see analytics. The
platform owner administers everything.

## Request flows (high level)

**Onboarding via poster QR:** poster QR вҶ’ app store вҶ’ app opens with
`?code=` deep link вҶ’ `GET /api/onboarding/:code` returns theme + business вҶ’ app
pre-selects them вҶ’ customer signs up вҶ’ adds pass to wallet.

**Add to wallet:** app calls `POST /api/passes {cardId, platform}` вҶ’ backend
mints `serial_number`, `lookup_token`, `auth_token` вҶ’ returns `.pkpass`
(Apple, if certs configured) or Google Wallet save link. Pass barcode/NFC
carries `lookup_token`.

**In-store redemption:** customer shows QR / taps NFC / reads code вҶ’ vendor
tablet or POS calls `POST /api/redeem` with the token + vendor + optional
purchase amount вҶ’ backend locks the discount row `FOR UPDATE`, validates all
rules (expiry, global/per-customer caps, city variations), records the
redemption + audit row, returns discount validity + amount вҶ’ **cashier applies
the discount manually**.

**Pass updates:** admin/vendor changes a discount вҶ’ backend updates the pass and
sends an APNs push (stub until Apple key configured) вҶ’ wallet re-fetches
`GET /api/passes/:serial`.
